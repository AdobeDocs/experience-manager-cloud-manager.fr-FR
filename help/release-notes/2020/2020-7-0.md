# Notes de mise à jour de la version 2020.7.0 {#release-notes-for}

La section ci-dessous présente les notes générales de mise à jour de la version 2020.7.0 de [!UICONTROL Cloud Manager].

## Date de publication {#release-date}

La date de publication de la version 2020.7.0 de [!UICONTROL Cloud Manager] est le vendredi 9 juillet 2020.

## Nouveautés {#whats-new}

* Désolidarisez et attachez les instances Dispatcher des équilibreurs de charge pendant les déploiements d’exploitation et votre environnement fonctionne désormais de manière plus cohérente.

* Le conteneur de création Cloud Manager prend désormais en charge Java 8 et Java 11.

* Les pipelines de Cloud Manager prennent désormais en charge les variables et les secrets définis par le client. Voir [Variables de pipeline](/help/getting-started/build-environment.md#pipeline-variables) pour plus d’informations.

## Correctifs {#bug-fixes}

* Les options **Annuler** et **Enregistrer** sur la page Modification du pipeline hors production n’étaient pas toujours visibles.

* Certaines erreurs liées au processus de qualité du code pouvaient entraîner la génération incorrecte du fichier journal.

* Certains journaux d’étape de pipeline volumineux ne pouvaient pas être téléchargés de manière cohérente via l’interface utilisateur.

## Problèmes connus {#known-issues}

* Si un environnement AMS contient une instance de secours, le message consigné indique que l’instance est hors service, et non en mode de secours.

* En raison d’un changement dans la façon dont la couverture de code est calculée, la version _minimale_ du plug-in Jacoco est désormais 0.7.5.201505241946 (publiée en mai 2015). Les clients qui référencent explicitement une ancienne version reçoivent un message d’erreur dans le processus Qualité du code .
